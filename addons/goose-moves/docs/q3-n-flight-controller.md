# Q3 + Flight Controller

The hybrid controller keeps Q3 movement responsible for contact-heavy movement
and uses the flight motor only for sustained free-air movement. Both motors
operate on the same `CharacterBody3D`, so the handoff is a change in physics
rules, not a teleport or velocity conversion.

The design goal is simple: use Q3 movement to build speed from surfaces, then
let flight spend that speed through lift, drag, flap impulses, and attitude
control.

## Mode Handoff

Flight activation is gated by intent, contact state, and speed:

```gdscript
flap_hold_time >= flight_hold_threshold
and no_surface_contact_time >= flight_no_contact_threshold
and velocity.length() >= flight_min_activation_speed
and not knocked_down
```

`no_surface_contact_time` treats floor, wall, ceiling, and slide collisions as
contact. That keeps the controller in Q3 while the body is still using geometry
for movement: stair climbing, wall scraping, ramp launch, crouch slide, water
movement, and ordinary jumping all remain Q3 behaviors.

When flight starts, the current view and takeoff velocity build the aircraft
basis. The nose is aligned toward the projected takeoff velocity when that is
safe; otherwise the controller falls back to an upright yaw-only basis. Velocity
is preserved either way.

Flight exits on any surface contact. The body snaps upright for Q3, keeps the
incoming velocity, restores grounded motion settings, and resumes Q3 collision
rules. High-speed body-bounce can intercept either mode and force a reflected
knockdown response.

## Ground Movement Principles

Ground movement is Q3-style projection acceleration. Input produces a
`wish_direction` and `wish_speed`; acceleration only fills the missing component
of velocity along that direction:

```gdscript
current_speed = velocity.dot(wish_direction)
add_speed = wish_speed - current_speed
velocity += wish_direction * min(acceleration * dt * wish_speed, add_speed)
```

The cap is on the projection, not the total vector. That is why off-axis input
can preserve or grow total speed while still respecting the local wish-speed
limit.

Friction is applied before acceleration on grounded frames:

```gdscript
drop = max(speed, stop_speed) * friction * dt
velocity *= max(speed - drop, 0.0) / speed
```

This makes friction a scale-down of current velocity, while acceleration is an
add-along-wishdir operation. Those two operators are the core ground feel:
friction decides how aggressively committed speed decays, and acceleration
decides how quickly input rebuilds or redirects it.

Air movement keeps the same projection-acceleration skeleton but changes the
coefficients and, in Warsow-style movement, adds horizontal air-control rotation.
That rotation bends horizontal velocity toward the forward/back wish direction
without adding speed by itself. Strafe acceleration adds speed; air control
spends steering authority.

## Stair And Ramp Movement

Stair handling is not a separate locomotion mode. It is a collision retry used
when horizontal motion would hit a blocking face:

1. Test the intended horizontal motion.
2. If it hits a non-walkable face, test raising the body by `step_height`.
3. Retry the same horizontal motion from the raised position.
4. Let normal floor handling settle the body afterward.

Stepping is skipped while moving upward. A riser inside the step height becomes
walkable traversal; a taller riser remains a wall. Because the retry preserves
the horizontal intent, stairs do not need special input rules.

Ramp behavior comes from the same plane clipping used by slide movement. On
walkable slopes, velocity is projected onto the floor plane and floor snap keeps
the body attached. On steep ramp contacts, the optional ramp-launch path can
preserve upward velocity generated by clipping into the plane, turning fast
horizontal motion into takeoff velocity.

## Force Application

Q3 movement is velocity-operator driven: friction, acceleration, jump, ramp
clip, wall jump, and water movement directly edit `velocity`. It does not model
mass or accumulated forces.

Flight movement is force-integrated:

```gdscript
total_force = gravity_force + aerodynamic_force + extra_drag_force
velocity += (total_force / mass) * dt
```

Flap is the exception. It is an instantaneous velocity impulse rather than a
continuous force:

```gdscript
axis = forward * cos(flap_angle) + up * sin(flap_angle)
velocity += axis.normalized() * flap_impulse_strength
```

That split is intentional. Q3 keeps the sharp response expected from arena
movement. Flight uses mass-scaled forces so lift and drag depend on speed,
orientation, and airframe tuning, while flap remains an explicit player-driven
burst.

## Collision Detection And Response

The controller relies on `CharacterBody3D` collision queries, but the response
policy differs by mode.

Q3 uses floor state, ground traces, slide collisions, and explicit step tests to
classify movement. Walkable contacts enable ground friction, floor projection,
floor snap, jumping, crouch slide, and water/wade variants. Non-walkable
contacts clip velocity against collision planes; optional features such as
steep-ramp launch and wall jump read those same contact normals.

Flight uses `move_and_slide()`, clips velocity out of contacted planes, and
applies Q3-style floor friction if a floor normal is found. The hybrid wrapper
then treats any collision as a flight-exit signal, because sustained flight is
defined as free-air movement.

Body-bounce scans slide collisions for incoming normal speed:

```gdscript
normal_speed = max(0.0, -impact_velocity.dot(collision_normal))
```

If the strongest impact exceeds the bounce threshold, velocity is reflected
around that normal, scaled by restitution, then clamped by the bounce speed cap
when the cap is above zero. The knockdown timer suppresses Q3 control and blocks
flight activation until it expires.

## Flight Aerodynamics

Flight attitude is written directly from pitch and roll commands. There is no
angular velocity or torque integration. Manual controls rotate around body axes;
camera fly-by-wire converts a camera target into pitch and roll inputs.

Aerodynamic angles are measured in body space from current velocity:

```gdscript
aoa = -atan2(local_velocity.y, forward_speed)
sideslip = atan2(local_velocity.x, forward_up_plane_speed)
```

Lift and drag use dynamic pressure, so they scale with speed squared:

```gdscript
dynamic_pressure = 0.5 * air_density * speed * speed
drag = -airflow_direction * dynamic_pressure * reference_area * drag_coefficient
lift = lift_axis * dynamic_pressure * reference_area * lift_coefficient
```

The lift and drag coefficients come from angle-of-attack tables. Lift is
perpendicular to the relative wind in the body's symmetry plane, not simply
body-up. That avoids sneaking drag into the lift vector and keeps drag in the
drag table.

Pitch is limited against the max-lift angle derived from the lift table when
airspeed is meaningful. This prevents command input from driving the body past
the useful lift peak every frame. Sideslip compensation is a local yaw
weathervane: it turns the nose toward the velocity projection in the body's yaw
plane, capped per physics frame.

## Tuning Map

Use these settings as design levers rather than isolated sliders.

### Q3 → Flight Gates

| Parameter | Increase to | Decrease to |
|---|---|---|
| `flight_hold_threshold` | Require clearer flight intent; reduce accidental activation | Make flight trigger from shorter flap holds |
| `flight_no_contact_threshold` | Demand cleaner airtime; prevent wall/ceiling scrape activation | Allow faster conversion after jumps and launches |
| `flight_min_activation_speed` | Reserve flight for high-speed takeoffs | Allow low-speed hops to enter flight |

### Ground Speed And Control

| Parameter | Increase to | Decrease to |
|---|---|---|
| `movement_mode` | Switch the Q3 rule set; use this before hand-tuning coefficients | Return to a simpler or less assisted rule set |
| `move_speed` | Raise the target projection speed for ground input | Lower normal run/wish speed |
| `ground_acceleration` | Make starts and redirects bite harder | Make ground input feel heavier |
| `air_acceleration` | Make airborne strafing add speed faster | Make air movement preserve momentum more than build it |
| `friction` | Shorten slides and punish missed jumps | Preserve speed longer on ground contact |
| `stop_speed` | Make low-speed stopping more decisive | Allow gentler coast near rest |
| `auto_jump` | Preserve speed through repeated jumps | Require timed jump presses |

### Verticality, Stairs, And Surfaces

| Parameter | Increase to | Decrease to |
|---|---|---|
| `gravity` | Make jumps shorter and landings faster | Extend airtime and soften arcs |
| `jump_velocity` | Raise takeoff height and initial flight-entry energy | Keep movement closer to the ground |
| `step_height` | Let the controller climb taller risers | Make small ledges block movement sooner |
| `max_slope_angle` | Treat steeper surfaces as walkable | Convert more slopes into walls/ramp-launch candidates |
| `ramp_launch` | Enable steep-slope velocity conversion | Keep steep contacts as ordinary slides |
| `wall_jump` | Add wall-normal takeoff options | Remove wall-derived vertical/horizontal boosts |
| `crouch_slide` | Add landing-armed speed carry | Keep crouch as stance/speed reduction only |

### Flight Energy

| Parameter | Increase to | Decrease to |
|---|---|---|
| `gravity_scale` | Make flight sink and require more lift/flap energy | Make flight floatier |
| `mass` | Make forces change velocity more slowly | Make lift, drag, and gravity alter velocity faster |
| `flap_impulse_strength` | Add stronger burst acceleration | Make flap more corrective than propulsive |
| `flap_impulse_angle` | Bias flap toward climb | Bias flap toward forward speed |
| `flap_cooldown` | Make flap timing more deliberate | Allow rapid flap chaining |

### Aerodynamic Feel

| Parameter | Increase to | Decrease to |
|---|---|---|
| `reference_area` | Strengthen both lift and coefficient-table drag | Make the airframe more ballistic |
| `extra_linear_drag_quadratic_coefficient` | Add speed-squared damping independent of AoA | Preserve high-speed glide energy |
| `sideslip_compensation` | Enable automatic yaw alignment to velocity | Allow stronger slip/skid behavior |
| `sideslip_compensation_max_yaw` | Weathervane into the airflow faster | Let lateral velocity persist longer |
| `camera_fly_by_wire` | Make camera aim command the airframe | Use manual pitch/roll inputs |
| `camera_fly_by_wire_target_distance` | Smooth target pursuit over farther points | Make target following respond to nearer aim points |
| `camera_fly_by_wire_pitch_window` | Use direct pitch over a wider forward cone | Prefer bank-and-pull turns except near straight ahead |

### Impact And Recovery

| Parameter | Increase to | Decrease to |
|---|---|---|
| `body_bounce` | Enable reflected high-speed impacts and knockdown | Keep impacts under normal collision response |
| `body_bounce_min_normal_speed` | Trigger bounce only on harder impacts | Make more collisions cause bounce/knockdown |
| `body_bounce_restitution` | Preserve more speed after the reflection | Bleed more speed on impact |
| `body_bounce_max_speed` | Allow faster post-bounce launches | Cap crash bounce speed more aggressively; `0` disables the cap |
| `body_bounce_knockdown_duration` | Make crashes cost more control time | Restore control faster |

## Practical Recipes

- **Earlier flight entry:** lower `flight_min_activation_speed`, lower
  `flight_no_contact_threshold`, and keep `flight_hold_threshold` low enough
  that intent still feels deliberate.
- **Surf-to-flight gameplay:** enable `ramp_launch`, keep `friction` low enough
  to preserve approach speed, and use a higher `flight_min_activation_speed` so
  only real launches convert.
- **Tighter arena ground feel:** raise `ground_acceleration`, raise `friction`,
  lower `stop_speed` only if low-speed stopping feels too abrupt.
- **Floatier gliding:** lower `gravity_scale`, lower `mass`, and avoid excessive
  `extra_linear_drag_quadratic_coefficient`.
- **Faster diving flight:** lower `reference_area` or raise extra quadratic drag
  carefully; then compensate with flap strength if recovery becomes too weak.
- **More aircraft-like turns:** keep `camera_fly_by_wire` enabled, increase
  `sideslip_compensation_max_yaw`, and tune `camera_fly_by_wire_pitch_window`
  so forward targets pitch directly while side targets bank-and-pull.
- **Hard crash penalties:** enable `body_bounce`, lower the impact threshold,
  lower restitution, and raise knockdown duration.
